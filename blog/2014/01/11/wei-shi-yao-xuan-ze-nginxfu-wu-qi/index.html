为什么选择nginx服务器

Jan 11th, 2014 | Comments

以前Apache一直是web静态服务器领域的老大哥，占有绝对的市场分额，但是这种局面随着nginx的出现而不断改变。

现在越来越多的大公司已经从apache迁移到了nginx，主要原因就是nginx的超高的性能，甚至远远超过apache.那么了解nginx的安装，部署基本成为了每一个web工程师的必修课(尤其是Rails工程师).

1. 通过源码安装Nginx

  • 从nginx官网下载源代码 (选用最新的stable版本，假设下载到本地~/Download目录)
  • sudo mv ~/Download/nginx-1.4.4.tar.gz /var/local
  • cd /var/local/ ; sudo tar -xvf nginx-1.4.4.tar.gz
  • sudo ./config
  • sudo make
  • sudo make install
  • cd /usr/local/nginx/

好了，如果没有报错的情况下的化，那么nginx已经安装成功了，并且默认安装到了/user/local/nginx目录.在/user/local/nginx目录下执行ls -l你将看到如下输出:

安装完成后，可以使用sudo ./sbin/nginx 启动nginx, 如果没有任何输入，那么表示启动成功，如果提示(98: Address already in use)之类的表示nginx启动的端口已经被占用,那么只需停掉该端口占用的进程或者修改nginx的监听的端口（默认80）就好。
